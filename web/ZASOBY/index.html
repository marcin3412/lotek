Autoryzacja czyli jak to wszystko dziala

  • 1. Klient wysyla zapytanie do serwera
  • 2. Serwer sprawdza czy zapytanie wymaga autoryzacji.
  • 3. Jezeli zapytanie wymaga autoryzacji sprawdza czy zapytanie zawiera autoryzacje. Jezeli nie to zwracany jest stosowny komunikat, jezeli tak wykonywana jest akcja.
  • 4. Serwer wykonuje akcje.
  • 5. Serwer zwraca dane.

Zapytania Jak tworzyc zapytania

  • Zapytania w AJAX

    • $.ajax({
    url: 'adres:port/metoda',
    type: 'GET',
    data: dane,
    success: function(data){
    console.log(data);
    }
    });

  • Adres aktualnie nie wiem jaki bedzie gdyz nie mamy serwera. Do testowania pewnie localhost:8080
  • Dane -> ?name=value&name2=value2 itd.
  • Testowanie metody :D adres:port/metoda?dane np. localhost:8080/game/show?game_id=1

Odpowiedz Jak wyglada odpowiedz

  • Odpowiedzi sa zawsze w formie JSON
  • Wszystkie odpowiedzi wysylane sa razem z naglowkiem HTTP, a ich tresc zwracana jest w formacie JSON. Kazda akcja zawiera pole "status", ktore informuje o powodzeniu wykonania akcji. W przypadku prawidlowego wykonania akcji, pole zwraca wartosc "OK". W przypadku blednego wykonania akcji, pole "status" zwraca wartosc "ERROR". Do odpowiedzi moze zostac dolaczone dodatkowe pole "info" zawierajace informacje o zaistnialym problemie.

Lista Metod

account/
  • register Rejestracja konta
  • confirm Potwierdzanie konta
  • login Logowanie/Autoryzacja
  • logout Wylogowanie
  • revu_pass Prosba o reset hasla
  • conf_pass Resetowanie hasla
user/
  • about Dane uzytkownika
  • edit Edycja danych uzytkownika
  • changePass Zmiana hasla
  • changeEmail Zmiana adresu email
ticket/
  • add Utworzenie losu (tylko przed wybrana kumulacja)
  • modf Edycja losu (tylko przed kumulacja)
  • del Usuwanie losu (tylko przed kumulacja)
  • show Zwraca los
  • showAll Zwraca liste losow
game/
  • next Zwraca informacje na temat nablizszej kumulacji
  • last Zwraca informacje na temat ostatniej kumulacji
  • list Zwraca zapisane kumulacje
  • show Zwraca informacje na temat wybranej kumulacji
stat/
  • top zwraca topke z wybranego zestawienia
server/
  • stat zwraca informacje na temat stanu servera ON/OFF

account/register

Metoda sluzy do rejestracji nowego uzytkownika na serwerze.

Parametry:
  • imie
  • nazwisko
  • haslo
  • email
Odpowiedz:
  • status - OK lub ERROR
  • msg - jezeli ERROR to dlaczego

account/confirm

Metoda sluzy do potwierdzania konta. Uzytkownik po prawidlowej rejestracji otrzymuje jednorazowy kod sluzacy do aktywacji jego konta na adres email. Ta metoda aktywuje konto

Parametry:
  • confirm_cod - kod aktywacyjny otrzymany w wiadomosci email.
Odpowiedz:
  • status - OK lub ERROR

account/login

Metoda sluzaca do logowania sie do serwera. Uzytkownik otrzymuje kod autoryzacyjny, ktory nalezy umieszczac jako paramet w metodach wymagajacych autoryzacji.

Parametry:
  • email
  • haslo
Odpowiedz:
  • status - OK lub ERROR
  • ssid - kod do autoryzacji metod, ktore tego wymagaja

account/logout

Metoda sluzy do wylogowania uzytkownika.

Parametry:
  • ssid - kod autoryzacyjny uzytkownika
Odpowiedz:
  • status - OK lub ERROR

account/rev_pass

Metoda wysyla do uzytkownika kod do resetowania hasla na andres email. Parametry w tej metodzie mozna wysylac zamiennie.

Parametry:
  • email
  • login
Odpowiedz:
  • status - OK lub ERROR

account/conf_pass

Metoda dzieki, ktorej uzytkownik moze zresetowac swoje haslo.

Parametry:
  • confirm_cod
  • password
Odpowiedz:
  • status - OK lub ERROR

user/about

  • email

  • nazwisko
  • email
  • konto_od
